이 문단은 p 태그를 이용해 작성되어진 텍스트입니다. 이 텍스트는 문장이 길어짐에 따라 자동으로 줄바꿈이 되는 경우는 줄의 간격이 좁지만, p 태그가 끝난 상태에서 다른 문단이 시작되는 것은 아래와 같이 줄의 간격이 좀 더 커지는 것이 p 태그(문단)의 특징입니다.

p 태그 이외에도 여러 태그가 있으며, 맨 위에서부터 h1, h2, p, p, table 태그가 적용되었습니다. 이 때, 주의할 점으로는 시작 태그와 끝 태그 안에 다른 태그를 넣을 수 있지만, 시작-끝 태그가 번갈아가면서 지정되어서는 안됩니다.

table를 태그를 이용한 표 생성하는 예시입니다.
table 자식으로 하나의 tr태그를 지정하여 하나의 행(가로)를 입력할 수 있으며
th 또는 td 태그를 통해 하나의 열(세로)를 생성할 수 있습니다.
여기서 th는 제목열을, td는 일반적인 열을 의미합니다.

HTML5의 버전으로 작성할 경우 th, td가 아닌
thead와 tbody 태그를 사용할 수 있습니다. 기존에는 제목열을 한 줄 밖에 사용을 못했지만
thead를 사용하면 제목열을 2줄 이상 사용할 수 있습니다!

이러한 thead와 tbody를 구분하여 사용하면 검색엔진에 노출이 더 잘되는 효과가 있습니다. 따라서, 사용하지 않아도 표시에는 문제가 없지만 가능하면 사용하는 것을 권장합니다.

form 태그 실습




남성 여성
봄 여름 가을 겨울

form 태그와 자식으로 input 태그를 입력해 여러 폼(입력창)을 만들 수 있습니다. 또한 type 속성으로 여러 입력값을 변경할 수 있습니다. 위의 예시에서는 각각 text, password, number, radio, checkbox 속성을 사용하였습니다.

radio 버튼을 입력할 때는 하나의 그룹으로 묶기 위해 name 속성을 사용하여 하나의 그룹으로 지정할 수 있으며, value 속성을 이용해 서버에 전송할 데이터를 지정할 수 있습니다. 즉, 변수명으로 name을, 변수 값으로 vlaue를 지정합니다.

checkbox는 서버에 값을 줄 때 배열 타입으로 넘겨집니다. 배열 타입으로 넘길 경우 radio 처럼 name과 value를 입력하면 됩니다. 그렇게 되면 배열명으로 name으로, 배열 속 데이터들로 value를 넘겨질 수 있습니다.

이러한 입력된 값은 가능하면 서버에 전송하기 전, JS를 사용해 타당성 검사를 하여 입력이 넘어가기 전 서버 통신을 차단하여 트래픽을 줄일 수 있습니다.

실제 현장에서 form 테그를 쓸 때는 아래처럼 table과 함께 자주 사용합니다. 앞 뒤로 글자가 오는 경우가 많은데, 간격이 맞지 않는 상황을 방지할 수 있기 때문입니다.

위처럼 텍스트에 label 태그와 함께 for 속성을 사용하면, 텍스트를 클릭시 해당 for 속성값과 같은 id 값을 가진 폼에 자동으로 포커스가 맞춰지게 됩니다. 이는 PC에서는 효과가 미미하지만 모바일 환경에서는 사용자환경에 꽤나 큰 영향을 주므로 여유가 된다면 사용하는 것을 권장합니다.

이 외에도 input의 타입에는 search, time, week, range 등 다양한 타입이 있으니 필요하다면 검색하여 적용하면 됩니다! 참고: W3school HTML Input Types
